The user is reporting a violation. This is serious.

## Process

1. **Acknowledge** what went wrong — specific, no hedging
2. **Reflect** on which principle was violated and why
3. **Read** `~/.pi/agent/APPEND_SYSTEM.md`
4. **Strengthen** the violated principle, or add a new one if none covers it
5. **Commit** to not repeating it

## Writing principles

- Read the manifesto first — match its voice
- Strengthen existing principles before adding new ones
- Direct language, no rhetorical flourishes or repetitive "X is not Y — it is Z" patterns
- Values, not procedures. "Scope is sacred" — not "When asked to do X, only touch files that Y"
- No code snippets, variable names, or step-by-step scenarios
- If a word can be cut, cut it
